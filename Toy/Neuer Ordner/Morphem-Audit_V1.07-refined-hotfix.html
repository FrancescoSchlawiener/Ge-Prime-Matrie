System Guide

A short guide to the tensor model, the index fields, and the morpheme library.

What this app does
It turns a word into exact tensor-like fingerprints and reconstructs the same word from those values.
How to use it
Type a word, collapse it, then reconstruct it. In the Morpheme Library you can add fragments, search, import, export, and edit them.
S — Substance
The exact prime-product fingerprint of the word. It is stored as a BigInt, so long words stay exact.
I — Permutation Index
The lexicographic rank of the prime multiset. It preserves ordering information.
N — Permutation Space
The number of valid permutations for the current multiset.
Prefix
Comes before the root, e.g. un-, ver-, re-.
Stem
The semantic core, e.g. stand, kauf, bewusst.
Suffix
Comes after the root, e.g. -ing, -ung, -heit.
Custom / Other
User-defined or fallback fragments for multilingual expansion.
Local storage
The library is stored locally in IndexedDB, so your custom morphemes survive reloads on the same device.
Import / Export
Export writes a versioned JSON file. Import supports Merge, Replace, and Only missing.
Why are some fragments red?
Red fragments are not currently matched to the library and are marked as collisions.
Can I add languages?
Yes. Enter one or more language codes in the language field, separated by commas.

Rules

Long words are handled as exact BigInt fingerprints, not modulo-reduced strings.
Morphemes are matched by a weighted graph search, not a greedy longest-prefix pass.
Known fragments stay green; unmatched fragments are marked as collisions.
Favorites, usage counts, and recent history are stored locally in the browser.
